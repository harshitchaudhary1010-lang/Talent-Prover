UPDATE 2: EXPIRED TASK BLOCKING FOR STUDENTS
================================================================================

WHAT WAS CHANGED:
-----------------
Students can NO LONGER access or submit to expired tasks.

HOW IT WORKS:
-------------
1. When a task has a deadline that has passed (deadline < today)
2. The task will NOT appear in the student's task list
3. If a student tries to access an expired task directly (via URL)
4. They will see an "Task Expired" message with the expiration date
5. They CANNOT submit work to expired tasks

WHERE CHANGES WERE MADE:
------------------------

✓ Dashboard/task.php
  - Added expiration check when viewing individual tasks
  - Shows "Task Expired" message with date if deadline has passed
  - Blocks submission form for expired tasks

✓ api/tasks.php
  - Updated task list API for students
  - Filters out expired tasks: AND (t.deadline IS NULL OR t.deadline >= CURDATE())
  - Company and admin can still see all tasks

✓ index.php
  - Homepage task listing now excludes expired tasks
  - Only shows active, non-expired tasks to visitors

✓ tasks.php
  - Public task browsing page filters expired tasks
  - Statistics only count non-expired tasks

✓ api/ai_match.php
  - AI skill matching only considers non-expired tasks
  - No matches will be shown for expired tasks

VISUAL INDICATORS:
------------------

EXAMPLES OF EXPIRED TASK BEHAVIOR:
===================================

Scenario 1: Task with deadline 2025-12-31 (past date)
- Student: CANNOT see in task list
- Student: CANNOT access task details
- Student: CANNOT submit work
- Company: CAN still see and manage
- Admin: CAN still see and manage

Scenario 2: Task with deadline 2026-12-31 (future date)
- Student: CAN see in task list
- Student: CAN access task details  
- Student: CAN submit work
- Everyone: Normal access

Scenario 3: Task with NO deadline (NULL)
- Student: CAN see in task list
- Student: CAN access task details
- Student: CAN submit work
- Task never expires

SQL QUERIES USED FOR FILTERING:
================================

For students only:
------------------
WHERE t.status = 'active' 
AND (t.deadline IS NULL OR t.deadline >= CURDATE())

This means:
- Include tasks with no deadline (NULL)
- Include tasks where deadline is today or in the future
- Exclude tasks where deadline is in the past

For companies and admins:
-------------------------
No filtering - they can see all tasks regardless of expiration
